Basic definitions

RDF triples

RDF: Resource Description Framework
Consists of three strings shown in the image, stored as such in memory when using it with Java

Streams

  • Triples come one by one
  • Analyze during the stream
Triples come available one by one, we don't have the full set from the start, but we do want to analyze the data that has passed by

Bloom filters

3 insertions with 3 hash functions over a 20 bit array

Reason for usage: very fast and minimal memory usage

False positives: not 100% prevention, but minimize risk

Exact details: see reference articles in report

Spout: outputs data from text and feedst to bolts

Bolt builder: creates bloom filters

Bolt prober: Searches from bloom filter

BuilderProber Bolt: create bloom filter for other bolts to probe and search from other’s

Query Decomposition

  • 1-Variable Join
  • 2-Variable Join
  • Multi-Variable Join

1-Variable Join

A one variable join: 3 objects and predicates should be the same, select subjects

1-Variable Join

SELECT author WHERE
Paper = kNN AND
Work = INRIA AND
diplome = Ph.D
Find the authors for paper kNN who works in INRIA and who has a Ph.D diplome

2-Variable Join

2-Variable Join

SELECT author, diplome WHERE
Paper = kNN AND
Work = ECP

Multi-Variable Join

Multi-Variable Join

SELECT author, work, diplome WHERE
Paper = kNN
to find the authors for paper kNN, and the place they work, and their diplome

1-Variable Join Topology

SELECT author WHERE Paper = kNN AND work = ECP AND diplome = Ph.D
find the authors for paper kNN who works in ECP and who has a Ph.D diplome.

2-Variable Join Topology

SELECT author,diplome WHERE Paper = kNN AND work = ECP
find the authors for paper kNN and their diplome who works in ECP

Multi-Variable Join Topology

SELECT authors,work,diplome WHERE Paper = kNN
find the authors for paper kNN, and the place they work, and their diplome

Sliding window Integration

Bolt builder sends one bloom filter at a time to the bolt prober

Sliding window Integration

Built prober stores bloom filters and process them in batches, sliding window = batch size. The size of the sliding window is based on different test results.
